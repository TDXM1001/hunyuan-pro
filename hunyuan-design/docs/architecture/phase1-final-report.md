## 🎯 测试步骤

### 1. 打开测试页面

访问 `http://localhost:5173/demos/table-test`

### 2. 基础功能测试（5 分钟）

1. 页面加载后应显示 10 条用户数据
2. 点击分页器切换到第 2 页
3. 在搜索框输入 "user1"，等待 300ms
4. 点击"重置"按钮
5. 点击"刷新"按钮

### 3. 缓存测试（3 分钟）

1. 切换到第 2 页
2. 再切回第 1 页
3. 应看到消息："数据从缓存加载" ✅
4. 观察右上角缓存统计
5. 点击"清空缓存"按钮

### 4. 刷新策略测试（5 分钟）

1. 点击"测试新增刷新" - 应回到第 1 页
2. 切换到第 3 页，点击"编辑" - 应保持第 3 页
3. 点击"删除"按钮 - 智能处理页码
4. 点击"手动刷新" - 清空所有缓存

### 5. 控制台验证（1 分钟）

打开浏览器控制台（F12），应看到：

## 📞 如果遇到问题

### 常见问题

**1. 页面 404**
- 检查路由是否正确添加
- 检查文件路径：`apps/web-ele/src/views/demos/table-test.vue`

**2. 类型错误**
- 运行 `pnpm install` 确保依赖安装
- 检查 `@vben/art-hooks` 是否在 workspace 中

**3. 导入错误**
- 确认导入路径：`import { useTable } from '@vben/art-hooks/table'`

**4. 开发服务器未启动**
- 运行 `pnpm dev:ele`
- 等待 Vite 编译完成
