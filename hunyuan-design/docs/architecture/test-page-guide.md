## 🎯 如何测试

### 访问测试页面

开发服务器启动后，访问：
```
http://localhost:5173/demos/table-test
```

### 测试步骤

#### 1. 基础功能测试

1. **数据加载**
   - 页面加载后自动显示 100 条模拟数据
   - 默认每页 10 条

2. **分页测试**
   - 点击分页器切换页码
   - 切换每页条数（10/20/30/50/100）
   - 观察控制台：应显示 "✅ 数据加载成功"

3. **搜索测试**
   - 在"用户名"输入框输入：`user1`
   - 等待 300ms（防抖）
   - 表格应显示匹配的用户（user1, user10-19, user100）

4. **重置测试**
   - 点击"重置"按钮
   - 搜索条件清空，回到第一页

#### 2. 缓存系统测试

1. **缓存命中**
   - 切换到第 2 页
   - 再切回第 1 页
   - 应显示消息："数据从缓存加载" ✅
   - 控制台显示："🎯 缓存命中"

2. **缓存统计**
   - 页面右上角显示缓存统计
   - "缓存: X 条" - 已缓存的请求数
   - "命中率: X avg hits" - 平均命中次数

3. **清空缓存测试**
   - 点击"清空缓存"按钮
   - 缓存统计归零
   - 再次切换页面会重新请求

#### 3. 刷新策略测试

1. **新增刷新（refreshCreate）**
   - 点击"测试新增刷新"
   - 应回到第一页
   - 清空分页缓存

2. **更新刷新（refreshUpdate）**
   - 切换到第 3 页
   - 点击某一行的"编辑"按钮
   - 应保持在第 3 页
   - 仅清空当前页缓存

3. **删除刷新（refreshRemove）**
   - 点击某一行的"删除"按钮
   - 智能处理页码（如果当前页为空，自动回到上一页）

4. **手动刷新（refreshData）**
   - 点击"刷新"按钮
   - 清空所有缓存
   - 重新获取当前页数据

#### 4. 移动端适配测试

1. 打开浏览器开发者工具
2. 切换到移动设备模式（F12 → Toggle Device Toolbar）
3. 观察分页器：
   - 宽度 > 1200px：显示 7 个页码
   - 宽度 < 1200px：显示 5 个页码
   - 宽度 < 768px：分页器变小（small 模式）

#### 5. 控制台日志验证

打开浏览器控制台，应看到：

```
[useTable] 数据已缓存
✅ 数据加载成功: {数据条数: 10, 响应: {...}}
🎯 缓存命中: {数据条数: 10}
[useTable] 缓存命中
[useTable] 清空所有缓存 - 手动刷新
```
