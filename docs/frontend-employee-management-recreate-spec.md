# 员工管理页前端规范说明

## 目标

在不改变员工管理业务流程、接口契约和核心交互语义的前提下，
将 `hunyuan-design/apps/hunyuan-system/src/views/system/employee/` 固化为组织架构类
树 + 列表页面的第一张样板页，并且严格服从现有前端规范文档，而不是做一次性的“局部美化”。

当前目标是：

- 以员工管理页作为第一个高保真样板页
- 固化页面整体视觉层级，而不是只记录某几个按钮
- 页面只保留员工业务展示差异
- 已确认稳定的基础量下沉到共享层或父级规范

## 当前评估

按当前实现和规范核对，员工管理页已经没有需要继续调整的结构性样式问题。

当前已稳定的点：

- 左树右表的双栏结构、宽度和间距已经固定
- 搜索表单的高度、label 中线、输入框和按钮高度已经统一
- 表格工具按钮、列设置弹层、显示设置弹层已经统一到共享 `ArtTableHeader`
- 分页采用共享 `ArtTable` 的左总数 + 右分页控件形态
- 操作列字体、角色标签、头像和行高已经收敛到中等密度

后续只需要在真实数据、不同角色权限、窄屏和空数据状态下做点验，不需要再改变当前页面规范。

## 约束

必须遵守以下规范文档：

- `docs/frontend-foundation-style-standard.md`
- `docs/frontend-list-table-page-standard.md`
- `docs/frontend-tree-list-page-standard.md`
- `docs/frontend-dialog-drawer-form-standard.md`

本次不做：

- 新增业务功能
- 修改接口参数
- 改变新增/编辑/删除/批量删除的语义
- 重构整套共享组件抽象

## 参考图的核心视觉特征

从参考图提炼出的主要特征如下：

### 1. 页面整体更“轻”

- 页面背景是低对比浅灰
- 卡片是白色，边框很轻
- 阴影弱甚至接近无阴影
- 页面不是“厚重后台块”，而是“轻表面 + 清晰分区”

### 2. 左树右表是完整双栏工作台

- 左侧机构树不只是一个附属筛选器，而是稳定的结构导航区
- 右侧操作区是主工作面
- 两列同时保持白卡片语义，视觉上属于同一工作台

### 3. 筛选区是横向、紧凑、低噪声的

- 标签、输入框、下拉框、查询/重置按钮都在同一水平节奏里
- 输入控件高度一致，当前基线为 `32px`
- 查询按钮是筛选区唯一强强调元素

### 4. 主工具栏有明确主次

- `新增员工` 是唯一主按钮锚点
- `批量删除` 是次按钮
- 右侧工具按钮是辅助工具，不应抢过左侧主操作

### 5. 表格区更接近企业级展示面

- 表头更轻，行分割更细
- 头像、姓名、角色标签、操作列都有更明确的信息层级
- 角色标签是柔和的语义色，而不是默认组件直出

### 6. 分页是完整的一组操作信息

- 左边显示总数
- 中间是分页器
- 右边是跳页输入或页码跳转
- 它属于表格工作区的一部分，而不是孤立挂在底部

## 当前页与目标图的主要差异

### 当前页已有的正确基础

- 双栏结构已经存在
- 搜索卡、表格卡、组织树卡已经拆分完成
- `ArtSearchPanel / ArtTableHeader / ArtTable / ArtOrgTree` 已可支撑业务
- 基础尺寸基线已经接近规范

### 当前页已经收敛的差异

#### 1. 页面层次

当前已经从“共享组件拼装页”收敛为完整的业务工作台：左树、搜索、工具栏、表格和分页都有稳定位置和密度。

#### 2. 机构树

机构树采用 `248px` 左列、`12px` body padding、`36px` 节点高度、`14px / 20px` 节点文字。

#### 3. 筛选区

筛选区采用 `32px` 控件高度，label、输入框、选择器和按钮在同一垂直中线。

#### 4. 工具栏

右侧工具按钮统一为 `32px x 32px`，列设置弹层为 `260px`，显示设置弹层为 `176px`。

#### 5. 表格内容表达

头像、姓名、角色标签和操作列已经形成员工列表的业务表达，不再依赖默认表格直出。

#### 6. 分页信息

分页采用 `showTotalSummary: true`，左侧显示总数，右侧保留每页条数、页码、上一页/下一页和跳页输入。

## 本次设计方案

### 方案选择

采用：

- 页面高保真优先
- 页面级样式允许增强
- 共享层只抽稳定基础量

不采用：

- 只改共享组件再逼近目标图
- 为了复用性牺牲页面还原度

原因：

- 员工管理页是树 + 列表 + 筛选 + 工具栏 + 表格 + 分页的复合样板页
- 参考图是一整套页面视觉语言，不是几个孤立组件
- 如果先被共享抽象束缚，还原会始终不到位

## 具体还原范围

### A. 页面骨架

保留双栏结构，但增强样板页工作台语义：

- 左树右表整体间距继续使用 `16px`
- 左列保持 `248px`
- 外层内容继续使用规范中的 `12px` 密集业务页内边距
- 搜索卡和表格卡 body padding 统一为 `16px`
- 卡片边框、白底、圆角统一回到规范基线

### B. 左侧机构树

目标：

- 更接近参考图的企业组织树观感

要调整的点：

- 标题行视觉层级
- 树节点选中态
- 节点 hover 态
- 节点缩进和图标对齐
- 节点区上下留白

边界：

- 不改变树数据结构
- 不改变点击行为
- 不改变树筛选语义

### C. 右侧筛选区

目标：

- 还原参考图中的横向筛选条

要调整的点：

- 标签与控件的横向节奏
- 输入框与下拉框宽度分配：关键字约 `216px`，选择器约 `168px`
- 按钮位置与对齐
- 查询按钮强调度
- 重置按钮轻量感

边界：

- 不改搜索字段集合
- 不改搜索提交行为

### D. 主工具栏

目标：

- 建立明确主次关系

规则：

- `新增员工` 是主按钮锚点
- `批量删除` 是次按钮
- 右侧工具组只承担辅助操作

要调整的点：

- 左右工具组距离
- 工具按钮尺寸、背景、边框、圆角、hover
- 主次按钮在同一行的视觉平衡

### E. 表格区

目标：

- 向参考图靠近的企业级表格展示面

要调整的点：

- 表头字重和背景感
- 行高和行分隔
- 头像从首字母占位向更真实视觉靠近
- 姓名列信息层级
- 角色标签柔化处理
- 操作列颜色权重统一

边界：

- 不重写数据表格组件
- 不改列语义，只调展示层

### F. 分页区

目标：

- 还原完整分页工作区，而不是简单页码条

需要具备：

- 总条数显示
- 每页条数选择
- 页码区
- 前往第几页输入/跳转
- 当前页 size 大于总记录数时，分页控件仍然保留

边界：

- 基于共享 `ArtTable` 分页能力表达
- 员工页只通过 `pagination-options` 启用，不做页面级分页重写

## 共享层与页面层边界

### 已反哺共享层的内容

- `ArtSearchPanel` 的 `32px` 搜索控件高度
- `ArtSearchPanel` 的稳定按钮宽度，避免 loading 抖动
- `ArtTableHeader` 的 `32px` 工具按钮
- `ArtTableHeader` 的列设置和显示设置弹层皮肤
- `ArtTable` 的 `showTotalSummary` 分页摘要能力
- `ArtTable` 的分页控件宽度和单页不隐藏规则
- 16px 工具组区块间距

### 保留在员工页页面层的内容

- 机构树业务样式
- 员工表格的头像/标签/操作列观感
- 特定于员工页的搜索区比例与表格视觉

## 落地顺序

### 第 1 步

后续组织架构类页面优先复用员工管理页的页面骨架和共享组件配置。

### 第 2 步

只有当新页面出现相同视觉问题时，才继续抽取共享能力。

### 第 3 步

如果新页面只是业务字段不同，保持页面级配置，不新增抽象。

## 验证标准

实现完成后至少验证以下内容：

1. 员工管理页结构顺序与参考图一致
2. 左树右表的双栏观感接近参考图
3. 筛选区高度、间距、按钮主次符合本规范
4. 主工具栏以 `新增员工` 为主视觉锚点
5. 表格头像、角色标签、分页整体气质接近参考图
6. 页面不出现浏览器级滚动条异常
7. `pnpm --dir hunyuan-design -F @hunyuan/system run typecheck` 通过
8. 若修改共享 `art-hooks`，`pnpm --dir hunyuan-design -F @vben/web-ele run typecheck` 也必须通过

## 风险与取舍

### 风险

- 继续把局部视觉问题写在员工页里，会造成后续页面重复修正。
- 继续把业务展示差异抽到共享层，会误伤普通表格页面。

### 取舍

- 稳定几何、控件、分页、弹层放共享层。
- 头像、角色标签、组织树业务视觉保留在员工页。
