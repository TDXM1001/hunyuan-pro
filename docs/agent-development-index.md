# Agent 开发文档索引与最低验证矩阵

## 1. 用途

本文件只解决两件事：帮助 Agent 快速找到当前领域事实源，并为不同风险的变更给出最低验证集合。它不是任务设计、实施计划、进度报告或新的审批关卡；具体目标、实现细节、端口和命令仍由链接的领域文档与运行手册所有。

只读取完成下一步所必需的内容。多个来源同时相关时，优先级为：用户指定的当前任务契约、对应领域当前基线或设计、相关代码与测试；只有需要真实运行时才读取运行手册。任务契约的新鲜度只核对文件状态、直接上游，以及是否有更新的当前基线或实现证据与其冲突；无冲突后停止扩展历史。入口缺失或疑似过期时，先从仓库和最近变更核实，不凭旧计划推断当前事实。

## 3. 最低验证矩阵

| 变更类型 | 最低证据 | 关闭与发布条件 |
| --- | --- | --- |
| 纯文档或非行为配置 | 差异、结构、链接、一致性、UTF-8 | 检查通过且未改变运行行为 |
| 局部行为或缺陷 | 失败复现或失败测试、聚焦测试、相关模块门禁 | 原问题有当次通过证据，无受影响回归 |
| 普通跨层交付 | 最短纵向主链、API/契约检查、相关后端与前端测试、必要运行证据 | 主路径可运行，受影响异常路径有证据 |
| 平台、公共语义、数据、安全或迁移 | 基线与影响扫描、上下游兼容或恢复、聚焦与相关回归、真实运行证据、追加式验收记录 | 所有必要门禁完成；任何必要缺口均为 `NOT_RELEASABLE` |
| 真实 UI 或运行联动 | 契约或类型检查、真实浏览器流程、网络与运行状态证据 | 用户可见主路径和关键失败路径均已证明 |

矩阵定义最低证据，不要求重复运行同一门禁。更高风险可以增加证据，低风险不得套用平台级全量流程。环境或无关基线阻塞必要证据时，应记录缺口、影响、替代证据和关闭条件；可以保存实现进度，但不能进入可发布基线或宣称模块关闭。

## 4. 维护规则

- 只维护当前有效入口和最低验证分类，不在本文件复制领域设计、动态命令、端口或任务进度。
- 新增入口必须替代真实的查找成本；已有领域文档可以承载时不再创建同类索引。
- 删除或替换入口前先修正所有引用，避免根规则指向失效事实源。
