## 1. 当前能力事实

- 控制面提供版本化来源系统、外部应用、员工映射、流程绑定和事件订阅；发布后冻结 Graph、连接器、签名与重试事实。
- 外部协议固定在 `/open/bpm/v1`，支持发起流程、读取任务详情和审批/拒绝；公共编号使用 `BP-*`、`BT-*`，不暴露 Flowable ID、BPMN XML 或内部表结构。
- 外部应用使用时间戳、nonce、请求体 HMAC 认证，并校验应用状态、凭据有效期、来源系统发布状态和 scope；人工任务仍必须通过外部员工映射和 M4 任务授权。
- 流程绑定只接受 `EQ`、`NE`、`GT`、`GTE`、`LT`、`LTE`、`IN` 类型化条件；无匹配和同优先级多匹配均失败关闭，不允许脚本或实时跨库查询。
- 终态业务事件按已发布订阅扇出为独立回调记录，冻结连接器版本、操作、签名密钥引用和重试策略；投递复用 M5 的重试、死信和人工补偿边界。
- 管理端业务接入配置工作台覆盖五类配置，并保持平台登录、菜单和按钮权限边界。

## 4. 安全、幂等与失败闭环

- 未签名的开放 API 请求返回平台 `NO_PERMISSION`（`30005`），证明请求到达应用认证边界；管理配置 API 未登录返回 `30007`，未绕过平台登录。
- HMAC 使用常量时间比较，并覆盖时间戳、nonce、HTTP method、完整 request target 和 body；应用/来源状态、过期时间和 scope 在进入业务命令前校验。密钥只接受 `env:` 引用，不落库明文。
- 外部发起按应用与 `requestId` 持久幂等，并校验相同 requestId 的请求内容；重复成功请求复用同一不透明 `BP-*`，处理中或失败事实不会重复创建实例。任务动作继续复用 M4 的任务版本和命令幂等边界。
- 员工映射通过 `BpmActorScope` 注入当前员工，之后仍执行现有详情字段裁剪、允许动作和任务授权，不允许应用身份代替员工审批。
- 每个订阅按来源系统隔离并形成独立投递事实；外部 payload 使用不透明 `BP-*` 且不含内部实例主键。失败按冻结策略退避，耗尽后进入死信，人工补偿复用既有回调处置入口，不回滚审批终态。

## 6. 完成定义对账

1. 通用申请与外部发起都进入 M3 审批对象和 M4 统一任务主链；真实 Flowable 兼容套件证明该主链可运行和恢复。
2. 业务字段与详情继续由 M3 契约配置，M6 的绑定和结果订阅可在工作台新增、编辑、发布，无需新增业务专用 BPM Service。
3. Hunyuan 与外部任务详情复用同一任务详情、员工授权和允许动作；外部返回相同版本快照的裁剪视图。
4. 发起由 `requestId`、任务由 M4 命令边界、事件由订阅加 `eventId` 唯一事实分层幂等。
5. 审批快照独立于来源可用性；结果投递具备冻结重试、死信和人工补偿事实。
6. 开放协议只返回 `BP-*`、`BT-*` 和 Hunyuan 业务字段，不暴露引擎或旧作者模型结构。
