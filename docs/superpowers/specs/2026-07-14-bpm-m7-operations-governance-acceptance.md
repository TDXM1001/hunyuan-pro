## 1. 当前能力事实

- 定时投影消费时间事件、外部等待、业务回调、业务命令和通知的结构化失败事实，形成统一异常工单；不读取 Flowable 表或 BPMN XML。
- 异常队列支持业务键、定义版本、处理人、状态、SLA、失败码、事件 ID、节点和组织过滤，并按组织部门负责人分配初始责任人。
- 重试、补偿和终止路由到 M4-M6 已有运行服务；底层动作失败不会推进 M7 工单状态，未登记安全恢复路径的动作失败关闭。命令先原子预占幂等键，再执行副作用，并追加操作者、原因、前后快照和失败原因。
- 终止、归档、跨组织查看使用独立权限；普通运营人员被限制在本人部门，查询、详情、指标、导出和处置使用同一组织范围。
- 指标按定义版本、节点、组织、日期和失败码统计 SLA、积压、处理时长与失败趋势；导出和审计只返回运营字段，不返回请求、响应或业务载荷快照。
- 归档同时校验在途状态、法定保留、保留期限、业务证据引用和迁移来源引用；默认保留策略为 365 天，并支持按定义和业务类型覆盖。已归档工单不会被同一失败事实自动重开。

## 5. 完成定义对账

1. 业务键、定义版本、处理人、状态、SLA、失败码和事件 ID 均进入统一查询契约和工作台。
2. 五类核心异常具备自动检测、Hunyuan 对象定位、组织范围、责任人、受控恢复或终止和追加审计。
3. 重试、补偿、终止与归档具有专属权限、原因、前后快照、操作者和数据库原子幂等预占。
4. SLA、积压、处理时长和失败趋势按定义版本、节点、组织、日期和失败码聚合。
5. 查询、导出和审计只使用 M7 投影与脱敏 VO；M7 不查询 Flowable 表，也不返回原始业务载荷。
6. 归档服务端强制检查在途、法定保留、期限、业务证据和迁移来源引用，不能由前端绕过。
