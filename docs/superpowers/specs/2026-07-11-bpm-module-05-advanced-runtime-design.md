# BPM 模块 M5：高级流程运行设计

- 日期：2026-07-11
- 重审日期：2026-07-12
- 状态：当前模块设计；旧时间事件实现作为状态机与测试资产
- 优先级：P2
- 总体蓝图：`docs/superpowers/specs/2026-07-11-bpm-enterprise-blueprint.md`
- 前置条件：M4 实例、命令、终态和 Outbox 稳定

## 1. 结论

M5 在 M4 可信审批运行时上增加时间、外部等待和流程组合能力，统一解决“不是由人工按钮立即推进”的场景。

M5 负责任务 SLA、提醒、延迟、受控外部调用/等待、事件相关、调用子流程、父子传播和恢复状态。连接器、凭据、来源系统和业务事件协议由 M6 登记，M5 只消费冻结引用。

## 2. 节点与运行事实

M5 向 M1 节点目录登记：

- SLA/边界超时策略：到期、提醒、升级、低风险自动终态。
- 延迟节点：固定时长、固定时间、受控业务日期事实。
- 外部动作/等待节点：登记连接器、操作、映射、等待模式、超时策略。
- 调用子流程节点：冻结子定义版本、输入输出映射和传播策略。

对应运行事实至少区分计划、等待、执行中、成功、可重试失败、人工处置、已取消和失效。每个事实拥有稳定事件键、实例/节点版本和审计时间线。

## 3. 端到端业务线

```text
发布冻结高级节点配置
-> M4 到达 authored 节点
-> 创建 timer / 外部命令 / 等待事实 / 子实例
-> 时间、回调或子流程结果相关
-> 校验当前节点与事件版本
-> 幂等推进 M4
-> 记录成功、失效、重试或补偿事实
```

父流程等待子流程时，输入映射、子定义版本、父子关系和输出映射必须结构化保存。父取消、子拒绝、超时和失败传播由节点策略明确配置。

## 4. 安全与失败闭环

- timer 与人工动作竞争时由 M4 当前任务/节点版本裁决，过期 timer 只记录失效事实。
- 回调必须验证应用、签名、相关键和等待版本；重复、乱序或无法关联的回调不能推进实例。
- 连接器失败使用持久命令、退避重试和人工重放；不在流程定义中保存 URL 或凭据。
- 服务重启后计划和等待事实可恢复，不依赖内存定时任务。
- 子流程失败不被隐式吞掉，按策略暂停父流程、拒绝、补偿或进入人工处置。
- 已发生的外部副作用不能伪装成本地事务回滚，必须由 M6 补偿契约或 M7 人工处置闭环。

## 5. 现有资产处理

旧 SLA、time event、external wait、HMAC、连接器安全和恢复测试可以吸收为新状态机断言；旧节点作者配置、实体表、服务 API 和运行投影不构成兼容要求。旧子流程设计中的版本冻结、映射和传播思想可吸收，但实现必须以正式 Graph 和新运行事实为输入。

## 6. 完成定义

1. 真实 Flowable timer 创建、到期、服务重启恢复和过期事件失效均通过。
2. SLA 提醒、转管理员和自动终态与人工动作竞态只形成一个有效结果。
3. 外部调用成功/失败、回调成功、重复/乱序/伪造回调和等待超时均有证据。
4. 父子流程启动、等待、输入输出、完成、拒绝、失败和取消传播端到端闭环。
5. 任一高级事件可从定义配置、运行事实、尝试记录和处置动作完整追踪。
6. 不存在回调直接改实例状态、任意 execution jump 或内存恢复旁路。
